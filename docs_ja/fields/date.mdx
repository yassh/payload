### Display Format and Picker Appearance

These properties only affect how the date is displayed in the UI. The full date is always stored in the format __PROTECTED_BLOCK_48__ (e.g. __PROTECTED_BLOCK_49__).

__PROTECTED_BLOCK_50__ determines how the date is presented in the field **cell**, you can pass any valid [unicode date format](__PROTECTED_BLOCK_68__).

__PROTECTED_BLOCK_51__ sets the appearance of the **react datepicker**, the options available are __PROTECTED_BLOCK_52__, __PROTECTED_BLOCK_53__, __PROTECTED_BLOCK_54__, and __PROTECTED_BLOCK_55__. By default, the datepicker will display __PROTECTED_BLOCK_56__.

When only __PROTECTED_BLOCK_57__ is set, an equivalent format will be rendered in the date field cell. To overwrite this format, set __PROTECTED_BLOCK_58__.

## Timezones

To enable timezone selection on a Date field, set the __PROTECTED_BLOCK_60__ property to __PROTECTED_BLOCK_61__:

__PROTECTED_BLOCK_7__

This will add a dropdown to the date picker that allows users to select a timezone. The selected timezone will be saved in the database along with the date in a new column named __PROTECTED_BLOCK_62__.

You can customise the available list of timezones in the [global admin config](../admin/overview#timezones).

<Banner type="info">
  **Good to know:**
  The date itself will be stored in UTC so it's up to you to handle the conversion to the user's timezone when displaying the date in your frontend.

Dates without a specific time are normalised to 12:00 in the selected timezone.

</Banner>
