フィールドアクセス制御は、ドキュメント内の特定の[フィールド](../fields/overview)へのアクセスを制限するために使用される[アクセス制御](../access-control/overview)です。

フィールドにアクセス制御を追加するには、[フィールド設定](../fields/overview)で`access`プロパティを使用します：

```ts
import type { Field } from 'payload'

export const FieldWithAccessControl: Field = {
  // ...
  access: {
    // highlight-line
    // ...
  },
}
```

<Banner type="warning">
  **注意：** フィールドアクセス制御は、[コレクションアクセス制御](./collections)のような[クエリ](../queries/overview)制約の返却をサポートしていません。
</Banner>

## 設定オプション

アクセス制御はリクエストの操作に固有です。

フィールドにアクセス制御を追加するには、[フィールド設定](../fields/overview)で`access`プロパティを使用します：

```ts
import type { CollectionConfig } from 'payload';

export const Posts: CollectionConfig = {
  slug: 'posts',
  fields: [
    {
      name: 'title',
      type: 'text',
      // highlight-start
      access: {
        create: ({ req: { user } }) => { ... },
        read: ({ req: { user } }) => { ... },
        update: ({ req: { user } }) => { ... },
      },
      // highlight-end
    };
  ],
};
```

以下のオプションが利用可能です：

| 関数         | 目的                                                                                                    |
| ------------ | ---------------------------------------------------------------------------------------------------------- |
| **`create`** | 新しいドキュメントを作成する際にフィールドの値を設定する能力を許可または拒否します。[詳細](#create)。 |
| **`read`**   | フィールドの値を読み取る能力を許可または拒否します。[詳細](#read)。                               |
| **`update`** | フィールドの値を更新する能力を許可または拒否します。[詳細](#update)。                            |

### Create

新しいドキュメントを作成する際にフィールドの値を設定する能力を許可または拒否するブール値を返します。`false`が返された場合、渡された値は破棄されます。

**利用可能な引数プロパティ：**

| オプション       | 説明                                                                                                                   |
| ---------------- | ----------------------------------------------------------------------------------------------------------------------------- |
| **`req`**        | 現在認証されている`user`を含む[Request](https://developer.mozilla.org/en-US/docs/Web/API/Request)オブジェクト |
| **`data`**       | ドキュメントの作成に渡される完全なデータ。                                                                                 |
| **`siblingData`** | ドキュメントの作成に渡される、直接隣接するフィールドデータ。                                                               |

### Read

フィールドの値を読み取る能力を許可または拒否するブール値を返します。`false`の場合、プロパティ全体が結果のドキュメントから省略されます。

**利用可能な引数プロパティ：**

| オプション       | 説明                                                                                                                   |
| ---------------- | ----------------------------------------------------------------------------------------------------------------------------- |
| **`req`**        | 現在認証されている`user`を含む[Request](https://developer.mozilla.org/en-US/docs/Web/API/Request)オブジェクト |
| **`id`**         | 読み取られているドキュメントの`id`                                                                                              |
| **`doc`**        | 完全なドキュメントデータ。                                                                                                      |
| **`siblingData`** | 読み取られているドキュメントの、直接隣接するフィールドデータ。                                                                  |

### Update

フィールドの値を更新する能力を許可または拒否するブール値を返します。`false`が返された場合、渡された値は破棄されます。

`false`が返され、フィールドの値を更新しようとした場合、操作はエラーを**投げません**が、フィールドは更新操作から省略され、値は変更されないままになります。

**利用可能な引数プロパティ：**

| オプション       | 説明                                                                                                                   |
| ---------------- | ----------------------------------------------------------------------------------------------------------------------------- |
| **`req`**        | 現在認証されている`user`を含む[Request](https://developer.mozilla.org/en-US/docs/Web/API/Request)オブジェクト |
| **`id`**         | 更新されているドキュメントの`id`                                                                                           |
| **`data`**       | ドキュメントの更新に渡される完全なデータ。                                                                                 |
| **`siblingData`** | 更新に渡される、直接隣接するフィールドデータ。                                                          |
| **`doc`**        | 更新が適用される前の、完全なドキュメントデータ。                                                                        |
