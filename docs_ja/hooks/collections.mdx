コレクションフックは、特定の[コレクション](../configuration/collections)内のドキュメントに対して実行される[フック](./overview)です。ドキュメントのライフサイクルの特定のイベント中に独自のロジックを実行できます。

コレクションにフックを追加するには、[コレクション設定](../configuration/collections)で`hooks`プロパティを使用します：

```ts
import type { CollectionConfig } from 'payload'

export const CollectionWithHooks: CollectionConfig = {
  // ...
  hooks: {
    // highlight-line
    // ...
  },
}
```

<Banner type="info">
  **ヒント：** フィールドレベルでフックを設定して、フックロジックを特定のフィールドに分離することもできます。[詳細](./fields)。
</Banner>

## 設定オプション

すべてのコレクションフックは[同期または非同期関数](./overview#async-vs-synchronous)の配列を受け入れます。各コレクションフックは、そのタイプに基づいて特定の引数を受け取り、特定の出力を変更する機能を持ちます。

```ts
import type { CollectionConfig } from 'payload';

export const CollectionWithHooks: CollectionConfig = {
  // ...
  // highlight-start
  hooks: {
    beforeOperation: [(args) => {...}],
    beforeValidate: [(args) => {...}],
    beforeDelete: [(args) => {...}],
    beforeChange: [(args) => {...}],
    beforeRead: [(args) => {...}],
    afterChange: [(args) => {...}],
    afterRead: [(args) => {...}],
    afterDelete: [(args) => {...}],
    afterOperation: [(args) => {...}],
    afterError: [(args) => {....}],

    // 認証有効フック
    beforeLogin: [(args) => {...}],
    afterLogin: [(args) => {...}],
    afterLogout: [(args) => {...}],
    afterRefresh: [(args) => {...}],
    afterMe: [(args) => {...}],
    afterForgotPassword: [(args) => {...}],
    refresh: [(args) => {...}],
    me: [(args) => {...}],
  },
  // highlight-end
}
```

### beforeOperation

`beforeOperation`フックは、操作が受け入れる引数を変更したり、操作が開始される前に実行される副作用を実行するために使用できます。

利用可能なコレクション操作には、`create`、`read`、`update`、`delete`、`login`、`refresh`、`forgotPassword`があります。

```ts
import type { CollectionBeforeOperationHook } from 'payload'

const beforeOperationHook: CollectionBeforeOperationHook = async ({
  args,
  operation,
  req,
}) => {
  return args // 必要に応じて変更された操作引数を返す
}
```

`beforeOperation`フックには以下の引数が提供されます：

| オプション       | 説明                                                                                                                   |
| ---------------- | ----------------------------------------------------------------------------------------------------------------------------- |
| **`collection`** | このフックが実行されている[コレクション](../configuration/collections)。                                                 |
| **`context`**    | フック間で渡されるカスタムコンテキスト。[詳細](./context)。                                                               |
| **`operation`**  | このフックが実行されている操作の名前。                                                                                   |
| **`req`**        | [Web Request](https://developer.mozilla.org/en-US/docs/Web/API/Request)オブジェクト。[Local API](../local-api/overview)操作ではモックされます。 |

### beforeValidate

`create`および`update`操作中に実行されます。このフックにより、受信データがサーバーサイドで検証される前にデータを追加または整形できます。

これはクライアントサイドの検証の前には実行されないことに注意してください。フロントエンドでカスタムフィールドコンポーネントをレンダリングし、`validate`関数を提供する場合、検証が実行される順序は次のとおりです：

1. `validate`がクライアントで実行される
2. 成功した場合、`beforeValidate`がサーバーで実行される
3. `validate`がサーバーで実行される

```ts
import type { CollectionBeforeValidateHook } from 'payload'

const beforeValidateHook: CollectionBeforeValidateHook = async ({ data }) => {
  return data
}
```

`beforeValidate`フックには以下の引数が提供されます：

| オプション        | 説明                                                                                                                   |
| ----------------- | ----------------------------------------------------------------------------------------------------------------------------- |
| **`collection`**  | このフックが実行されている[コレクション](../configuration/collections)。                                                 |
| **`context`**     | フック間で渡されるカスタムコンテキスト。[詳細](./context)。                                                               |
| **`data`**        | 操作を通じて渡される受信データ。                                                                                       |
| **`operation`**   | このフックが実行されている操作の名前。                                                                                   |
| **`originalDoc`** | 変更が適用される前のドキュメント。                                                                                      |
| **`req`**         | [Web Request](https://developer.mozilla.org/en-US/docs/Web/API/Request)オブジェクト。[Local API](../local-api/overview)操作ではモックされます。 |

### beforeChange

検証の直前に、beforeChangeフックはcreateおよびupdate操作中に実行されます。この段階では、データは未検証のユーザー入力として扱われるべきです。必須フィールドが存在するか、フィールドが正しい形式であるかの保証はありません。そのため、このデータを副作用に使用するには手動検証が必要です。保存されるデータの形状をオプションで変更できます。

```ts

  return data
}
```

### beforeRead

```ts

  return doc
}
```

| **`query`**      | リクエストの[クエリ](../queries/overview)。                                                                                      |
| **`req`**        | [Web Request](https://developer.mozilla.org/en-US/docs/Web/API/Request)オブジェクト。[Local API](../local-api/overview)操作ではモックされます。 |

### afterOperation

`afterOperation`フックは、操作の結果を変更したり、操作が完了した後に実行される副作用を実行するために使用できます。

利用可能なコレクション操作には、`create`、`find`、`findByID`、`update`、`updateByID`、`delete`、`deleteByID`、`login`、`refresh`、`forgotPassword`があります。

```ts
import type { CollectionAfterOperationHook } from 'payload'

const afterOperationHook: CollectionAfterOperationHook = async ({ result }) => {
  return result
}
```

`afterOperation`フックには以下の引数が提供されます：

| オプション       | 説明                                                                                                                   |
| ---------------- | ----------------------------------------------------------------------------------------------------------------------------- |
| **`args`**       | 操作に渡される引数。                                                                                                              |
| **`collection`** | このフックが実行されている[コレクション](../configuration/collections)。                                                 |
| **`req`**        | [Web Request](https://developer.mozilla.org/en-US/docs/Web/API/Request)オブジェクト。[Local API](../local-api/overview)操作ではモックされます。 |
| **`operation`**  | このフックが実行されている操作の名前。                                                                                           |
| **`result`**     | 変更前の操作の結果。                                                                                                    |

### afterError

`afterError`フックは、Payloadアプリケーションでエラーが発生したときにトリガーされます。これは、エラーをサードパーティサービスにログ記録したり、開発チームにメールを送信したり、エラーをSentryやDataDogにログ記録するなどに役立ちます。出力は結果オブジェクト/ステータスコードを変換するために使用できます。

```ts
import type { CollectionAfterErrorHook } from 'payload';

const afterErrorHook: CollectionAfterErrorHook = async ({
  req,
  id,
  doc,
}) => {...}
```

`afterError`フックには以下の引数が提供されます：

| 引数                | 説明                                                                                                                                                                                     |
| ------------------- | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **`error`**         | 発生したエラー。                                                                                                                                                                        |
| **`context`**       | フック間で渡されるカスタムコンテキスト。[詳細](./context)。                                                                                                                                 |
| **`graphqlResult`** | GraphQLコンテキスト内でフックが実行される場合に利用可能なGraphQL結果オブジェクト。                                                                                                          |
| **`req`**           | 現在認証されている`user`と Local API インスタンス`payload`を含む`PayloadRequest`オブジェクト（[Web Request](https://developer.mozilla.org/en-US/docs/Web/API/Request)を拡張）。 |
| **`collection`**    | このフックが実行されている[コレクション](../configuration/collections)。                                                                                                           |
| **`result`**        | RESTコンテキストからフックが実行される場合に利用可能な、フォーマットされたエラー結果オブジェクト。                                                                                       |

## TypeScript

Payloadは各コレクションフックの型をエクスポートしており、以下のようにアクセスできます：

```ts

  CollectionBeforeOperationHook,
  CollectionBeforeValidateHook,
  CollectionBeforeChangeHook,
  CollectionAfterChangeHook,
  CollectionAfterReadHook,
  CollectionBeforeReadHook,
  CollectionBeforeDeleteHook,
  CollectionAfterDeleteHook,
  CollectionBeforeLoginHook,
  CollectionAfterLoginHook,
  CollectionAfterLogoutHook,
  CollectionAfterRefreshHook,
  CollectionAfterMeHook,
  CollectionAfterForgotPasswordHook,
  CollectionRefreshHook,
  CollectionMeHook,
} from 'payload'
```
